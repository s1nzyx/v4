│   ├── action_register.php  # Обработка регистрации и хэширования

│
├── config.php               # Инициализация сессии и подключение к БД (PDO)
├── index.php                # Главная страница (Категории, Краткая информация)
├── product_list.php         # Каталог товаров с сортировкой (Цена/Популярность)

└── logout.php               # Скрипт уничтожения сессии (Выход)


2. Архитектура Базы Данных (СУБД MySQL)

База данных спроектирована с учетом поддержки целостности данных (каскадные удаления `ON DELETE CASCADE` и зануление внешних ключей `ON DELETE SET NULL`).

Для совместимости со старыми версиями MySQL/MariaDB длины уникальных полей оптимизированы до **191 символа** (обход ошибки `#1071`).

Схема связей таблиц:

`categories.id` $\rightarrow$ `products.category_id` *(Один ко Многим)*
`users.id` $\rightarrow$ `orders.user_id` *(Один ко Многим, необязательная связь для гостей)*
`orders.id` $\rightarrow$ `order_items.order_id` *(Один ко Многим)*
`products.id` $\rightarrow$ `order_items.product_id` *(Один ко Многим)*

Описание таблиц:

users: Хранит данные пользователей. Пароли захешированы через `PASSWORD_DEFAULT`.
categories: Категории сувениров (Народные, Статуэтки и др.) с уникальным символьным кодом.
products: Номенклатура товаров со ссылкой на ID категории, ценой, именем файла картинки и счетчиком популярности.
orders: Заказы. Содержат как уникальный генерируемый номер `order_number` (вид: ORD-64A...), так и полную контактную информацию на случай, если заказ оформляет неавторизованный гость.
order_items: Связующая таблица «Многие ко Многим» между заказами и товарами. Фиксирует историческую цену товара на момент покупки.

3. Описание функциональных модулей

3.1. Ядро системы (`config.php`)

Централизованный файл конфигурации. Автоматически запускает механизм сессий `session_start()` на каждой странице. Подключение к СУБД реализовано через интерфейс **PDO** в режиме вывода исключений (`PDO::ATTR_ERRMODE => PDO::ERRMODE_EXCEPTION`). Это гарантирует безопасность и отказоустойчивость.

3.2. Авторизация и Регистрация

Реализована по сессионной модели:

1. **Регистрация**: Обработчик проверяет уникальность введенного Email. Если адрес свободен, пароль хэшируется, создается запись в БД, а данные пользователя пишутся в `$_SESSION['user']` для автоматического входа.
2. **Авторизация**: Сверяет Email и проверяет валидность хэша пароля функцией `password_verify()`. При успехе перенаправляет на Главную.
3. **Безопасность**: Все ошибки валидации передаются обратно через временные переменные сессии `$_SESSION['auth_error']` и сбрасываются сразу после отображения.

3.3. Каталог и Сортировка (`product_list.php`)

Скрипт динамически реагирует на GET-параметры в URL:

* При переходе по категориям фильтрует товары (`category_id`).
* При клике на кнопки сортировки модифицирует SQL-запрос на лету: добавляет `ORDER BY price ASC` (по цене) или `ORDER BY popularity DESC` (по популярности).

3.4. Модуль корзины и Оформить заказ

Корзина (`$_SESSION['basket']`): Представляет собой ассоциативный массив, где *ключ — это ID товара*, а *значение — его количество* в корзине. Это позволило избежать лишних запросов к БД при изменении состава корзины.
Оформление заказа (`order.php`): Запись данных происходит в рамках **базы данных транзакций** (`$pdo->beginTransaction()`). Если запись самого заказа прошла успешно, но на этапе записи его позиций (в `order_items`) произошел сбой, транзакция полностью откатывается (`$pdo->rollBack()`), предотвращая появление "битых" заказов без товаров.

3.5. Личный кабинет (`profile.php`)

Страница закрыта от гостей: если сессия пуста, срабатывает редирект на `login.php`. Для авторизованного пользователя выгружается список его заказов из таблицы `orders` с автоматической конвертацией дат к человеческому формату (`d.m.Y`) и выводом статуса заказа (`open` — Открыт / `closed` — Закрыт).

4. Инструкция по развертыванию (Деплой)

1. **Создание БД**: Открой **phpMyAdmin**, перейди во вкладку «SQL» и выполни скрипт из Шага 3 (создание базы `comfy_shop` и наполнение таблиц).
2. **Перенос файлов**: Скопируй папку проекта в директорию вашего локального сервера (например, `C:\OSPanel\domains\comfy\`).
3. **Настройка подключения**: Если у твоего локального MySQL установлены нестандартный пароль или порт, открой файл `config.php` в корне проекта и отредактируй строку подключения:
```php
$dsn = "mysql:host=localhost;dbname=comfy_shop;charset=utf8mb4";

```


4. Загрузка медиа: Помести фоновые изображения и картинки товаров в папку `assets/images/`, назвав их строго в нижнем регистре: `souvenir1.png`, `souvenir2.png` и т.д.
5. Тестирование: Открой браузер и перейди по адресу `http://comfy/`. Начни с регистрации нового пользователя.
